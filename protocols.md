# Протоколы

**Что делать при каждом старте — проекта, сессии, задачи**

---

## Старт нового проекта

1. **Прочитать ядро wiki** — principles/, rules/, decisions/
2. **Определить группу проекта** — UI/DS, Flow, утилита, заготовка...
3. **Создать папку** в `projects/` с README.md, requirements.md, progress.md
4. **Прописать связи** в ecosystem-map.md (откуда берём, куда даём)
5. **Добавить в SUMMARY.md**
6. **Коммит и пуш**

## Старт новой сессии (AI-песочница)

1. **Прочитать worklog.md** — что делали в предыдущих сессиях
2. **Прочитать progress.md** проекта — где остановились
3. **Клонировать исходники** — shallow clone нужных репо
4. **Прочитать файлы перед изменением** — не читал — не утверждаешь
5. **Делать по 4D:** Door → Do → Document → Double-check
6. **Записывать в worklog.md** после каждого логического шага

## Старт новой задачи

1. **Прочитать требования** — requirements.md проекта
2. **Проверить что не нарушаешь принципы** — anti-hallucination, anti-monolith
3. **Прочитать исходные файлы** — реальные, не из памяти
4. **Сделать** — код, изменения
5. **Проверить** — git diff, build, lint
6. **Записать** — progress.md, worklog.md

## Правило каскада

Нельзя пропустить шаг. Не прочитал — не делаешь. Не проверил — не коммитишь. Не записал — не переходишь к следующему.
