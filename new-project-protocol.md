# Протокол нового проекта

**При создании нового репозитория в экосистеме.**

---

## Шаг 1: Определить группу

По [Карте экосистемы](ecosystem-map.md) — куда относится проект:
UI / Flow / Код-галерея / Цвет / CSS / Обучение / Портфолио / Пакет / Утилита / Заготовка

## Шаг 1.5: Найти похожий проект

Перед созданием — проверить, нет ли уже ACTIVE или NEW проекта с похожей функциональностью. Если есть:
- Можно ли расширить существующий вместо создания нового?
- Если нет — зафиксировать почему (это будет первый ADR)

Примеры крупных ACTIVE-проектов: P-MAS-architector (multi-agent), 3A Studio (IDE), UI-Kit (дизайн-система), Component-Browser (каталог).

## Шаг 2: Создать репо

```
1. Создать на GitHub (stsgs1980/)
2. Определить статус: [NEW]
3. Выбрать стек (по decisions/language-strategy.md: код = English)
4. Настроить .gitignore, README.md
```

## Шаг 3: Зарегистрировать в Wiki

```
1. Добавить в ecosystem-map.md — в правильную группу
2. Добавить в projects/index.md — с описанием и ссылкой
3. При необходимости — создать projects/<name>/README.md
4. Обновить SUMMARY.md
5. Запушить Wiki
```

## Шаг 4: Принципы и правила

Новый проект автоматически подчиняется:
- [Анти-галлюцинация](principles/anti-hallucination.md) — не читал — не утверждаешь
- [Строительство 4D](principles/construction-4d.md) — Door → Do → Document → Double-check
- [Верификация](principles/verification.md) — сказал "сделано" — докажи
- [Языковая стратегия](decisions/language-strategy.md) — код English, wiki русский

## Шаг 5: Проектные решения

Если проект требует специфических решений (БД, стек, архитектура) — создать ADR:
- Кросс-проектные → `decisions/`
- Проект-специфичные → `projects/<name>/decisions/`

---

## Чек-лист

```
□ Группа определена
□ Проверено: нет похожего ACTIVE/NEW проекта (или зафиксировано почему новый)
□ Репо создан на GitHub
□ Статус [NEW] присвоен
□ Добавлен в ecosystem-map.md
□ Добавлен в projects/index.md
□ SUMMARY.md обновлён
□ Wiki запушена
```
