## Зачем {#зачем}

Песочница Z.ai = временный контейнер. Когда сессия заканчивается -- всё стирается. Поэтому:
- Все решения хранятся в **StsDev-Wiki** (GitHub)
- Весь код хранится в **3A Studio** (GitHub)
- Песочница = только рабочее место на пару часов

---

## Важные ограничения песочницы {#ограничения}

| Правило | Почему |
|---------|--------|
| Рабочая папка ТОЛЬКО `/home/z/my-project/` | Dev-сервер песочницы привязан к этому пути |
| НЕ клонировать в подпапку (`/home/z/my-project/my-repo/`) | Dev-сервер не увидит проект |
| МОЖНО клонировать ПРЯМО в `/home/z/my-project/` | Перезаписывает заготовку, dev-сервер подхватит |
| НЕ запускать `npm run dev` / `bun run dev` вручную | Сервер управляется песочницей автоматически |
| Git push нужен для сохранения | Без push всё пропадёт при закрытии сессии |
| Песочница НЕ имеет gh CLI | Нужен PAT токен для git push/pull |
| Сессия может оборваться в любой момент | Всегда коммить и пушить промежуточные результаты |

## Сценарий A: Новая песочница -- начинаем с нуля {#сценарий-a}

### Шаг 1: Скажи агенту что делать

Первое сообщение в новой песочнице:

```
Привет. Я работаю над проектом 3A Studio.
Инструкции и контекст -- в StsDev-Wiki.

Сделай следующее:
1. git clone https://github.com/stsgs1980/StsDev-Wiki.git /tmp/wiki
2. Прочитай projects/3a-studio-master-plan.md
3. Прочитай decisions/ -- все решения
4. Расскажи что ты понял и продолжи с того места где остановились
```

### Шаг 2: Клонируем проект (если код уже существует)

```bash
# Удаляем заготовку песочницы
rm -rf /home/z/my-project/src /home/z/my-project/package.json /home/z/my-project/tsconfig.json

# Клонируем 3A Studio ПРЯМО в рабочую папку
git clone https://github.com/stsgs1980/3a-studio.git /tmp/3a-studio
cp -r /tmp/3a-studio/* /home/z/my-project/
cp /tmp/3a-studio/.gitignore /home/z/my-project/ 2>/dev/null

# Устанавливаем зависимости
cd /home/z/my-project && bun install

# Dev-сервер песочницы подхватит автоматически
```

### Шаг 3: Если 3A Studio ещё не создан (Phase 0)

Тогда работаем как обычно -- пишем код в `/home/z/my-project/` и периодически делаем git init + push.

## Сценарий B: Сессия оборвалась -- продолжаем {#сценарий-b}

### Что произошло

Сессия закрылась. Код не запушен. Всё потеряно в песочнице. НО:

- Если код был **запушен в GitHub** -- ничего не потеряно
- Если код **НЕ был запушен** -- потеряно (кроме wiki)

### Как продолжить

```
1. Открыть новую песочницу
2. Написать: "Продолжаю работу над 3A Studio. Последний коммит: <хэш>. Читай wiki."
3. Агент: git clone wiki -> читает контекст -> git clone 3a-studio -> продолжает
```

### ПРАВИЛО: Коммить часто

```bash
# После каждого значимого изменения
git add -A
git commit -m "feat: описание что сделал"
git push origin main
```

Минимум -- коммит перед тем как отойти от компьютера.

### Если git push не работает (нет токена)

1. Зайди на GitHub -> Settings -> Developer settings -> Personal access tokens
2. Создай токен с правом `repo`
3. Дай агенту: `git remote set-url origin https://<ТОКЕН>@github.com/stsgs1980/3a-studio.git`

## Сценарий C: Клонируем StsDev-Wiki для чтения {#сценарий-c}

Wiki клонируем В ОТДЕЛЬНУЮ ПАПКУ (не в /home/z/my-project/):

```bash
# Wiki -- только для чтения контекста
git clone https://github.com/stsgs1980/StsDev-Wiki.git /tmp/wiki

# НЕ в /home/z/my-project/StsDev-Wiki/ -- это сломает dev-сервер
```

Агент читает файлы из `/tmp/wiki/`, но пишет код в `/home/z/my-project/`.

## Чек-лист перед завершением сессии {#чек-лист}

Перед тем как закрыть песочницу:

- [ ] `git add -A && git status` -- проверить что все изменения включены
- [ ] `git commit -m "описание"` -- закоммитить
- [ ] `git push origin main` -- запушить (если не пушится -- всё потеряно!)
- [ ] Если wiki обновлялась: `cd /tmp/wiki && git add -A && git commit -m "..." && git push`
- [ ] Написать в чат: "Остановился на Phase X, задача Y. Следующий шаг: Z"

## Частые проблемы и решения {#проблемы}

### "git push просит пароль/логин"

Песочница не имеет gh CLI. Нужен PAT токен:
```
git remote set-url origin https://<ТОКЕН>@github.com/stsgs1980/<РЕПО>.git
```

### "Port 3000 already in use"

Не запускай dev-сервер вручную. Песочница управляет им сама.

### "Cannot find module"

```bash
cd /home/z/my-project && bun add <package>
```

### "Превью не работает"

1. Проверь логи: `cat /home/z/my-project/.zscripts/dev.log | tail -20`
2. Проверь линтер: `bun run lint`
3. Перезапуск через init (ТОЛЬКО если проект ещё не клонирован):
   `curl https://z-cdn.chatglm.cn/fullstack/init-fullstack_1775040338514.sh | bash`

### "Сессия оборвалась, код не запушен"

К сожалению, код потерян. Но:
- Wiki на GitHub -- сохранена
- Предыдущие коммиты 3A Studio на GitHub -- сохранены
- Нужно пересоздать только то что делал в последней сессии

### "Хочу работать с двумя репо одновременно"

```bash
# 3A Studio -- в рабочей папке (для dev-сервера)
cd /home/z/my-project/

# StsDev-Wiki -- в /tmp/ (только чтение)
git clone https://github.com/stsgs1980/StsDev-Wiki.git /tmp/wiki

# P-mas-studio -- в /tmp/ (только чтение, для копирования кода)
git clone https://github.com/stsgs1980/P-mas-studio.git /tmp/p-mas-studio
```

### "Инструкция из песочницы говорит НЕ клонировать"

Та инструкция (ZAI_SANDBOX_INSTRUCTIONS.md) написана для **простых одностраничных проектов**. Для 3A Studio (monorepo, 12 экранов, 4 пакета) она не подходит. Клонируем ПРЯМО в /home/z/my-project/, не в подпапку.

6. Скопируй код в /home/z/my-project/ и запусти

Если кода ещё нет -- начни Phase 0 из Master Plan.
