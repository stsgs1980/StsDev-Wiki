# Decision: Протокол строительства 4D

**Дата:** 2026-05-30
**Статус:** Принято
**Проблема:** Сессии обрываются, контекст теряется, работа не доезжает до конца

---

## Суть

Каждая единица работы проходит 4 шага. Ни один не пропускается.

**Door → Do → Document → Double-check**

---

## 1. Door — Вход в сессию

Перед любой работой — обязательные действия:

```
1. Прочитать /home/z/my-project/3A-PROGRESS.md
2. Прочитать /home/z/my-project/worklog.md
3. Найти последний незавершённый этап
4. Спросить: "Продолжаем с этапа X?"
```

Без этого — код не пишется. Сначала контекст, потом действия.

---

## 2. Do — Делать

Одна задача = один этап = одна фича.

```
- Прочитать ВСЕ затронутые файлы ПЕРЕД изменением
- Написать код
- Запустить проверку (build / lint / typecheck)
```

Правило: **не прочитал файл — не пишу в него.**

---

## 3. Document — Зафиксировать

После каждого этапа — три обязательных действия:

**3A-PROGRESS.md** (карта где мы):
```
## Этап N: [название]
- Статус: DONE / IN-PROGRESS / BLOCKED
- Что сделано: [конкретно]
- Что изменилось: [файлы]
- Как проверить: [команда]
- Следующий этап: N+1 — [название]
```

**worklog.md** (детали):
```
Task ID: N
Agent: [кто]
Work Log: [шаги]
Stage Summary: [результат]
```

**Git commit** (доказательство):
```
git add + commit с префиксом этапа
```

Без записи в 3A-PROGRESS.md — этап не считается завершённым.

---

## 4. Double-check — Перепроверить

Перед статусом DONE — ответить на 3 вопроса:

```
1. Файлы реально изменились? → git diff
2. Сборка проходит?          → npm run build / npx tsc
3. Не сломал предыдущее?     → перечитать затронутые файлы
```

Без трёх YES — этап остаётся IN-PROGRESS.

---

## Файл 3A-PROGRESS.md — пульт управления

Не worklog (он для деталей). Это — карта где мы:

```
## Карта этапов 3A Studio

### Фаза 1: Фундамент
[1] SQLite/PostgreSQL фикс ........... DONE
[2] Skill model: SkillFile ........... IN-PROGRESS
[3] API: CRUD для SkillFile .......... TODO

### Фаза 2: Экспорт
[4] Export: DB → SKILL.md ............ TODO
[5] Export: DB → .zip ................ TODO
[6] zai deploy bridge ................ TODO

### Фаза 3: Валидация
[7] Prompt scoring в пайплайне ....... TODO
[8] Тесты: минимальный набор ........ TODO
[9] CI: validate на каждый коммит .... TODO
```

Любая сессия открывает этот файл и видит: где мы, что дальше, что сломано.

---

## Каскад — почему это доедет до конца

Каждый этап блокируется предыдущим:

```
[1] schema фикс → блокирует [2] SkillFile → блокирует [3] API → блокирует [4] Export
```

Нельзя:
- Делать Export (4) без SkillFile (2)
- Делать zai deploy (6) без Export (4)
- Делать тесты (8) без работающего фундамента (1-3)

Если этап не DONE — следующий не начинается.

---

## Защита от потери работы

| Ситуация | Что происходит | Как восстанавливаем |
|----------|---------------|-------------------|
| Сессия оборвалась | Код на диске, процессы мертвы | `git pull` + читать 3A-PROGRESS.md |
| Код не запушен | Потерян | Переписать по worklog + 3A-PROGRESS |
| Забыли что делали | Следующая сессия не знает | 3A-PROGRESS.md = источник истины |
| Две сессии параллельно | Конфликт в общих файлах | Станислав решает, кто первый |

---

## Почему это работает

1. **Door** — нет работы вслепую, всегда знаем где мы
2. **Do** — одна задача за раз, не распыляемся
3. **Document** — следующая сессия может продолжить без Станислава
4. **Double-check** — не объявляем готовым то что не проверено
5. **Каскад** — нельзя прыгнуть вперёд, не закончив предыдущее

Это не теория — это механизм который держит строительство на рельсах между сессиями.
