## Инструменты верификации

[verify-docs](tools/verify-docs/README.md) — data-driven проверка консистентности документации. Реализует принцип [Верификация](principles/verification.md): сверяет структуру wiki (проекты, принципы, ссылки) с реальным состоянием файловой системы. Конфигурация: `verify-docs.json` + `verify-docs.plugins.ts`.

```bash
bun run tools/verify-docs/src/cli.ts    # ручной запуск
```

## Как пользоваться

1. `pull.bat` — подтянуть свежую версию с GitHub
2. Редактируешь любой `.md` файл
3. `push.bat` — **автоматически запускает verify-docs**, затем отправляет изменения

Если создаёшь **новый** файл — пропиши его в `SUMMARY.md`. Если файла нет в `SUMMARY.md` — GitBook его не увидит.

> **verify-docs встроен в `push.bat`** — каждая отправка автоматически проверяет консистентность документации. Если есть проблемы — push блокируется. Для обхода: `git push --no-verify`.
>
> **Linux/Mac:** запусти `bash setup-hooks.sh` один раз после clone — установит git pre-push hook для автоматической проверки при любом `git push`.

## Статусы проектов

| Статус | Значение |
|--------|----------|
| [ACTIVE] | Рабочий код, активно развивается |
| [NEW] | Greenfield, начало разработки |
| [REFERENCE] | Read-only, не модифицируем |
| [LOCAL] | Локальный исходник |
| [ARCHIVED] | Мёртвый, не развивается |
| [DELETED] | Удалён с GitHub |
| [PACKAGE] | Переиспользуемый пакет |
| [EMPTY] | Пустой репозиторий |
